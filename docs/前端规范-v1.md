# GeWeHub 前端规范 v1

> 状态：定稿（2026-07-06）
> 适用：`web/` 全部代码。本文档是 `开发计划-v1.md` M4 的强制约束，验收时逐条对照。
> 原则：克制、一致、信息密度优先。这是给自己用的专业工具，不是营销页——禁止炫技，禁止"AI 味"装饰。

## 一、设计基调（先读这个）

GeWeHub 的视觉参照两个对象：**微信桌面端**（聊天工作台部分）和 **Linear / NewAPI 风格的现代控制台**（管理页部分）。整体气质：安静、紧凑、灰白底、低饱和、几乎无装饰。

**绝对禁止清单（出现即验收不通过）：**

- 渐变背景、渐变按钮、渐变文字（headers 上尤其禁止 `bg-gradient-to-r`）
- 大圆角卡片堆叠（radius 超过 12px 的容器）、玻璃拟态（backdrop-blur 卡片）、彩色大阴影
- emoji 当图标用（图标一律 lucide-react）
- 每个卡片/按钮不同的圆角、不同的间距、不同的灰色
- 彩虹色系的统计卡片（一排卡片每个一个颜色）
- 无意义的动画：页面进场动画、卡片 hover 上浮 + 放大、骨架屏闪烁过快
- 居中大标题 + 大段描述文案的"落地页式"页面头
- `dark:` 双主题——第一版只做浅色主题，不写任何 dark 变体（避免维护两套没人验收的样式）

## 二、设计 Token（唯一来源）

所有颜色、圆角、阴影必须走 CSS 变量 / Tailwind 语义类，**禁止在业务组件里写裸值**（如 `text-[#666]`、`rounded-[10px]`、`bg-gray-450`）。

### 2.1 颜色

采用 shadcn/ui 默认 neutral 体系，主色定为微信绿的克制版本：

```css
:root {
  --background: 0 0% 100%;
  --foreground: 240 10% 10%;          /* 主文字，近黑 */
  --muted: 240 5% 96%;                /* 浅灰底：侧栏、输入框底 */
  --muted-foreground: 240 4% 46%;     /* 次要文字 */
  --border: 240 6% 90%;
  --primary: 142 70% 38%;             /* 微信绿 #1da750 附近，仅用于主按钮/激活态/自己的气泡 */
  --primary-foreground: 0 0% 100%;
  --destructive: 0 72% 51%;
  --ring: 142 70% 38%;
  --radius: 0.5rem;                   /* 全局唯一圆角基准 8px */
}
```

用色规则：

- 页面 95% 的面积只允许出现：白、`muted` 灰、`border` 灰、两级文字灰黑。
- `primary` 绿只出现在：主操作按钮、选中/激活态、自己发送的消息气泡、开关 on 态。一屏内绿色元素不超过 3 处。
- 语义色只用于状态 Badge 和告警（见 5.3），不得用于装饰。
- 图表/头像占位等需要多色时，用低饱和固定色板（tailwind 的 `*-100` 底 + `*-700` 字），不许随机彩色。

### 2.2 字体与字号

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC",
             "Hiragino Sans GB", "Microsoft YaHei", sans-serif;
/* 等宽（JSON/ID/代码）： */
font-family: ui-monospace, SFMono-Regular, Menlo, Consolas, monospace;
```

字号只允许 5 档，禁止其他值：

| 用途 | 类 |
|---|---|
| 页面标题（每页一个） | `text-lg font-semibold`（18px） |
| 区块/卡片标题 | `text-sm font-medium`（14px） |
| 正文、消息文本、表格 | `text-sm`（14px） |
| 次要信息、时间戳、meta | `text-xs text-muted-foreground`（12px） |
| ID/JSON/代码 | `text-xs font-mono` |

禁止 `text-2xl` 以上出现在任何管理页（这不是营销站）。数字大屏（观测页统计值）最大 `text-2xl font-semibold`。

### 2.3 间距

只用 4px 基准的 Tailwind 刻度：`1 2 3 4 6 8`（4/8/12/16/24/32px）。禁止 `p-5`、`p-7`、`m-9`、任意 `p-[13px]`。

固定习语（全站统一，不许每页自创）：

- 页面内容区 padding：`p-6`
- 卡片内 padding：`p-4`
- 表单字段纵向间隔：`space-y-4`
- 行内图标与文字间隔：`gap-2`
- 列表项内 padding：`px-3 py-2`

### 2.4 圆角与阴影

- 圆角只有三档：`rounded-md`（输入框/按钮）、`rounded-lg`（卡片/弹窗）、`rounded-full`（头像/Badge 圆点）。
- 阴影只有两档：`shadow-sm`（卡片，可选，优先用 border 分隔）、弹窗/下拉用 shadcn 默认。禁止自定义彩色阴影。
- 分隔优先用 `border`，不靠阴影和留白猜。

## 三、布局规范

### 3.1 全局框架

```text
┌──────┬─────────────────────────────────────┐
│      │  顶栏 h-12：面包屑/页标题   右侧：GeWe状态点+管理员菜单 │
│ 侧栏 ├─────────────────────────────────────┤
│ w-56 │  内容区（聊天工作台 或 管理页）        │
│      │                                     │
└──────┴─────────────────────────────────────┘
```

- 左侧全局导航栏固定 `w-56`，`bg-muted/40` + 右 border；导航项：图标 16px + 文字 `text-sm`，激活态 `bg-muted text-foreground font-medium`（不用绿色大色块）。
- 导航顺序：聊天工作台 / 应用管理 / 账号与联系人 / 推送日志 / 发送记录 / 运行观测 / 接入设置。
- 内容区管理页最大宽度 `max-w-6xl mx-auto`；聊天工作台占满全宽。

### 3.2 聊天工作台三栏

```text
│会话列表 w-72│ 消息区 flex-1（min-w-0）│ 详情栏 w-80（可收起）│
```

- 三栏均独立滚动（`overflow-y-auto`），页面本身不滚动（`h-screen` 布局）。
- 会话列表项：头像 40px + 两行文本（名称 `text-sm` / 最近消息 `text-xs text-muted-foreground truncate`）+ 右上角时间 `text-xs`；选中态 `bg-muted`；未绑定应用的会话不加特殊颜色，已绑定的在名称后跟应用 Badge（`variant=secondary`，`text-xs`）。
- 详情栏可整体收起（右上角按钮），收起后消息区自动占满。

### 3.3 消息气泡（重点，照微信桌面端）

- 消息区背景 `bg-muted/30`（微信的浅灰底）。
- 对方消息靠左：头像 32px + 白底气泡 `bg-background border rounded-lg`；自己消息靠右：`bg-primary/90 text-primary-foreground` 气泡，头像在右。
- 群聊对方消息在气泡上方显示发送者名（`text-xs text-muted-foreground`，含成员备注时显示"备注(昵称)"）。
- 气泡最大宽度 `max-w-[65%]`，文本 `text-sm leading-relaxed whitespace-pre-wrap break-words`。
- 时间分隔条：两条消息间隔 >5 分钟时居中显示 `text-xs text-muted-foreground`。
- 各类型渲染：
  - image：缩略图 `max-w-60 max-h-60 rounded-md`，点击 Dialog 看大图；
  - voice：内联 audio 播放条 + 时长文本；
  - file：文件图标 + 文件名 + 大小 + 下载按钮，卡片式 `border rounded-md p-3 w-64`；
  - link / mini_program：标题 + 描述两行截断 + 缩略图右置的卡片 `w-72`；
  - chat_record：标题 + 前三条摘要 + "查看 N 条记录"，点击 Dialog 内嵌套渲染条目（条目内复用同一套节点渲染组件——组件必须递归设计）；
  - quote：气泡内顶部一个 `bg-muted rounded px-2 py-1 text-xs text-muted-foreground border-l-2` 的引用块；
  - system：居中灰色小字条，无气泡无头像；
  - unsupported：气泡内 `[暂不支持的消息类型: rawType]` 灰字；
  - revoked：原气泡替换为居中灰条"某某撤回了一条消息"，右键仍可看原文。
- 消息右键菜单用 shadcn `ContextMenu`：查看详情 / 复制文本 / 复制标准 JSON / 查看原始 payload。

## 四、组件规范

1. **一律使用 shadcn/ui 既有组件**：Button、Input、Select、Dialog、Sheet(抽屉)、Tabs、Badge、Table、Tooltip、ContextMenu、DropdownMenu、Toast(sonner)、Switch、Skeleton。禁止引入第二套组件库（antd/mui/arco 等），禁止对同一交互重复造轮子。
2. 按钮：一屏内只有一个 `default`（绿色）主按钮；其余用 `outline`/`ghost`；危险操作 `destructive` + 二次确认 Dialog（确认文案写明后果，如"解绑后该会话消息将停止投递"）。按钮内图标 16px（`size-4`），图标+文字必须 `gap-2`。
3. 表格：一律 TanStack Table + shadcn Table 皮肤。必备：列头排序（可排序列）、分页器（每页 20）、右侧操作列（`ghost` 图标按钮 + Tooltip）。单元格内 ID 类内容用 `font-mono text-xs` + 点击复制。
4. 表单：React Hook Form + zod；label 在上（`text-sm font-medium`），错误信息 `text-xs text-destructive` 在字段下方；提交按钮 loading 态必须有（`disabled + Loader2 动画图标`）。
5. 复制交互全站统一：点击 → 图标短暂变 Check → toast "已复制"。封装一个 `<CopyButton>` 复用，禁止各处手写。
6. JSON 查看器：统一封装 `<JsonViewer>`——`bg-muted rounded-md p-3 text-xs font-mono overflow-auto max-h-96` + 右上角 CopyButton。原始 payload / 标准 JSON / GeWe 响应都用它。

## 五、状态设计（AI 最常偷懒的地方，逐条验收）

### 5.1 加载

- 列表/表格首屏：Skeleton（行状，3~5 条），禁止全屏 spinner。
- 局部刷新：保留旧数据 + 右上角小 spinner（TanStack Query `isFetching`），禁止整页闪白。
- 按钮触发的动作：按钮自身 loading，禁止全屏遮罩。

### 5.2 空状态

每个列表页必须有设计过的空状态：居中，lucide 图标 24px `text-muted-foreground` + 一句说明 `text-sm text-muted-foreground` + （若适用）主操作按钮。文案要具体：

- 会话列表空："暂无会话，收到第一条微信消息后会自动出现"
- 应用列表空："还没有应用，创建应用后可将会话绑定给它" + [新建应用]
- 推送日志空："暂无投递记录"

### 5.3 状态 Badge 色板（全站唯一映射，封装 `<StatusBadge>`）

| 语义 | 样式 | 用于 |
|---|---|---|
| 成功/在线/已投递/acked | `bg-green-100 text-green-700` | online、delivered、sent |
| 进行中/等待 | `bg-blue-100 text-blue-700` | queued、delivering、pending |
| 警告/降级 | `bg-amber-100 text-amber-700` | unknown、left、inactive |
| 失败/离线/dead | `bg-red-100 text-red-700` | failed、offline、dead |
| 中性/关闭/已撤回 | `bg-muted text-muted-foreground` | disabled、revoked、skipped |

Badge 一律 `text-xs rounded-full px-2 py-0.5`，不带图标（状态点场景用 `size-2 rounded-full` 圆点 + 文字）。

### 5.4 错误

- 接口错误：toast 显示 server 返回的 message（中文），不弹裸 HTTP 状态码。
- 页面级错误（路由 errorComponent）：图标 + "加载失败" + 错误摘要 + [重试] 按钮。
- 表单错误：字段级显示，不聚合成顶部大红条。

### 5.5 灰显（软状态数据）

已退群成员/已删除联系人：整行 `opacity-60` + 状态 Badge，头像叠加灰色滤镜。选择器（下拉选联系人/成员）默认过滤非 active，顶部提供"显示全部"开关。

## 六、交互细节

1. 所有 icon-only 按钮必须有 Tooltip。
2. 所有破坏性操作（解绑、重置 token、撤回消息）必须二次确认。
3. 长 ID（wxid、messageId、token）展示规则：`font-mono text-xs` + 中间省略（`abc…xyz`）+ hover Tooltip 全文 + 点击复制。
4. 时间显示：今天的显示 `HH:mm`，今年的显示 `MM-DD HH:mm`，更早显示 `YYYY-MM-DD`；hover Tooltip 显示完整时间。封装 `<TimeText>`。
5. 聊天区滚动：进入会话滚到底部；向上翻页加载历史时保持滚动位置不跳（记录 scrollHeight 差值）；新消息到达时若用户不在底部，不强制滚动，显示"↓ 新消息"浮标。
6. token/密码类字段默认打码显示（`••••`），提供眼睛切换 + 复制。
7. 键盘：发送框 Enter 发送、Shift+Enter 换行；Dialog 支持 Esc 关闭。

## 七、代码组织规范

1. 目录按 feature 分（见开发计划第一节），组件文件 PascalCase，hooks `use-xxx.ts`。
2. **API 层统一封装**：`lib/api.ts` 一个 fetch 包装（自动带 credentials、统一错误解析、401 跳登录）；每个 feature 一个 `queries.ts` 集中定义 query keys 和 hooks，**禁止组件内裸写 fetch**。
3. 类型一律从 `packages/contracts` import，**禁止在 web 内重复手写服务端类型**。
4. 组件拆分标准：超过 150 行或有独立复用价值才拆；不许过度拆成"一个 div 一个文件"。
5. 消息节点渲染必须是一个递归组件 `<MessageNodeView node={} depth={}>`，chat_record 条目、quote 内部都复用它，禁止为每个嵌套场景复制渲染代码。
6. 禁止 `any`（contracts 类型不够时先补 contracts）；禁止 `// @ts-ignore`。
7. 禁止引入未在技术选型中的依赖（moment、lodash、axios、redux 等）。日期用原生 `Intl`/自写 3 行工具，请求用 fetch。

## 八、验收方式

M4 验收时对照本文档执行三类检查：

1. **禁止项扫描**：grep 检查 `bg-gradient`、`dark:`、`text-[#`、`rounded-[`、`p-5`、`@ts-ignore`、`any`、非法依赖——零命中。
2. **截图对照**：聊天工作台、应用管理、推送日志、观测页四屏截图，对照第一节基调和第三节布局逐项核对（气泡样式、三栏宽度、Badge 色板、空状态）。
3. **交互走查**：第五、六节逐条人工走查（加载态、空态、错误态、复制、Tooltip、二次确认、滚动行为）。

## 九、专业化升级补充规范（2026-07-07 追加）

本章节深化前八章要求；若与前文冲突，以本章节为准。设计判断锚点：聊天工作台对照微信桌面端和 Chatwoot，管理控制台对照 Linear、Supabase Studio、Vercel Dashboard、shadcn/ui dashboard/tasks 示例。任何界面决策都必须能解释“Linear/Chatwoot 会怎么做”。

### 9.1 动效系统

动效只用于表达空间关系和状态变化，不做装饰。全站只能使用以下 token：

```css
:root {
  --duration-fast: 120ms;
  --duration-base: 180ms;
  --duration-slow: 240ms;
  --ease-out: cubic-bezier(0.16, 1, 0.3, 1);
  --ease-in: cubic-bezier(0.7, 0, 0.84, 0);
}
```

- 可动画属性仅限 `opacity`、`transform`、`background-color`、`border-color`、`box-shadow`；禁止对 `width/height/top/left` 做动画。
- hover 反馈使用 `--duration-fast` 的背景/边框过渡；按钮按压允许 `active:scale-[0.98]`。
- Dialog/Sheet/Popover/Tooltip 使用 Radix/shadcn 默认空间动效；Sheet 右侧滑入时长为 `--duration-slow`。
- 新消息进入最多一次 120ms fade-in，无位移；删除不做退场动画。
- Skeleton 只用同步 `animate-pulse`，禁止同屏多个不同节奏闪烁。
- 全局尊重 `prefers-reduced-motion: reduce`，降级为无动画、无滚动平滑。
- hover 信息卡 Popover 延迟 300ms 打开、100ms 关闭；TooltipProvider 使用 `delayDuration=400`、`skipDelayDuration=100`。

### 9.2 表格与管理型列表

管理型记录页必须采用 TanStack Table + shadcn Table + 服务端分页/排序/筛选。

- 列类型：文本左对齐；数字右对齐并 `tabular-nums`；时间统一 `<TimeText>`（相对时间 + Tooltip 绝对时间）；ID 统一 `font-mono text-xs` 中间省略 + 点击复制；状态统一 `<StatusBadge>`；微信实体统一 `<EntityCell>`（头像 24px + 名称 + 次行备注/wxid）。
- 列宽：时间 160px、状态 100px、操作 80px 定宽；主列弹性 `min-w-0 truncate`，超长内容必须 Tooltip 全文。表格容器可横向滚动，操作列 `sticky right-0` 并带左侧渐隐遮罩。
- 工具栏三段式：左侧关键字搜索（300ms 防抖、清空按钮）；中间 Facet filter（状态/类型多选，下拉项带计数，选中项显示可删除 chip）；右侧列显隐 View options 与刷新按钮（`isFetching` 时图标旋转）。
- URL 同步：筛选、排序、分页、选中行必须同步 searchParams；刷新和分享链接不丢状态。重置按钮一键清空全部筛选。
- 推送日志和发送记录当前采用服务端分页 + 状态筛选；URL 参数固定为 `status`、`page`，请求参数固定为 `take`、`skip`、`status`，默认每页 20 条。
- 推送日志默认聚焦失败投递：无 `status` 参数时等价于 `status=failed`，首屏必须先回答“有没有失败投递”；显式查看全部投递时 URL 使用 `status=all`，请求不带 `status` 参数，刷新和分享链接不丢“全部”语义。
- 行交互：整行 hover `bg-muted/50`；点击打开详情 Sheet；行内复制、外链、按钮必须 `stopPropagation`。
- 批量选择：支持时首列 checkbox + 表头半选态；选中后表格上方出现批量操作条：“已选 N 项 | 批量重试 | 取消”。
- 分页器：右下显示“共 N 条”、每页 20/50/100、首页/上一页/下一页/末页、第 x / y 页。若后端只能游标分页，页面必须明示为“上一页/下一页游标模式”。

### 9.3 详情、弹层与 JSON

- 详情统一 `<DescriptionList>`：两列网格，label `text-xs text-muted-foreground` 宽 120px，value `text-sm`；分组用 Separator + 组标题；空值显示 `—`。
- 详情 Sheet 头部固定：实体图标/头像 + 名称 + StatusBadge + 关闭按钮；正文滚动；底部固定操作栏。
- 关联数据用 Tabs：投递详情 = 基本信息 / 投递 payload / 重试历史；消息调试 = 概览 / 标准 JSON / 原始 payload / 投递记录。
- `<JsonViewer>` 必须升级为：语法高亮、默认展开 2 层、可折叠、hover 行高亮、单行复制路径、顶部工具条（复制全文 / 展开全部 / 折叠全部 / 搜索键名）。路径复制统一采用 `$` 根路径、对象点路径、数组下标格式（如 `$.payload.items[0].text`），特殊 key 使用 bracket 形式。
- 长文本字段（错误信息、XML、raw 文本）使用 `max-h-40 overflow-auto font-mono text-xs`，并提供 CopyButton。
- 图片大图和 chat_record 展开必须用 Dialog 无边距变体；chat_record 内部递归使用同一套消息节点渲染。

### 9.4 聊天工作台深化

- 消息分组：同一发送者 3 分钟内连续消息合并为组；首条显示头像和名称，后续悬挂缩进对齐，hover 才显示时间；组间距 16px、组内距 4px。
- 日期分隔：跨天显示居中日期条（今天/昨天/具体日期）。第一版会话打开至少定位到底部。
- 未读分隔线：若有未读字段，显示“以下是新消息”细线并定位到该处；字段未落库前可暂不展示，但实现需预留。
- 媒体占位：图片/视频按 payload width/height 设置 `aspect-ratio`；无尺寸时固定 200x150，占位加载完成后 fade-in，避免消息流 CLS。
- 发送状态机：发送后立即插入本地乐观消息（半透明 + 时钟），成功替换正式消息，失败显示红色感叹号并支持重试/删除。输入框提交后立即清空，不因发送中阻塞继续输入。
- 消息 hover 工具条：时间、复制、调试、引用回复预留；投递状态用气泡角落小圆点（queued 蓝 / delivered 绿 / failed 红），hover 见详情，点击打开消息调试投递 Tab。
- 会话列表：预留置顶分区；未读 Badge（>99 显示 99+）；最近消息摘要含发送者前缀；时间在右上；静音/异常状态图标位；支持键盘 ↑/↓ 切换会话。
- 输入区：textarea 自适应 1-6 行；粘贴图片进入待发送附件条；拖拽文件到消息区显示投放层；附件条显示文件名/大小/删除。
- 语音发送：主按钮文案固定为“录制语音”，点击必须直接开启麦克风录制，停止后提交发送；文件选择只能作为“语音发送选项”下拉中的兜底入口，不得作为语音主按钮默认行为。
- 右侧详情栏：分段折叠面板（会话信息 / 绑定与投递 / 成员群 / 快捷调试入口），折叠状态写入 localStorage。

### 9.5 Mutation 与 Query 策略

TanStack Query 默认策略：

| 数据 | staleTime | refetch 策略 |
|---|---:|---|
| 管理型列表 | 30s | 列表页聚焦可刷新，保留旧数据 |
| 微信实体信息（联系人/成员） | 5min | 按 wxid 或 groupId 缓存，禁止 N+1 |
| 运行观测 | 30s | `refetchInterval=30s` |
| 推送日志 / 失败任务 | 30s | 页面可见时刷新 |
| 聊天消息 | SSE 优先 | 当前会话增量追加，必要时精确补拉 |

mutation 影响范围必须精确：

| mutation | 更新方式 | 影响 query key |
|---|---|---|
| 登录/退出 | setQueryData | `auth.me` |
| 会话备注 | 乐观更新 + 失败回滚 | `workbench.workspace` |
| 会话绑定/解绑 | pending 态 + 成功刷新 | `workbench.workspace` |
| 群成员备注 | 乐观更新 + 失败回滚 | `workbench.group-members(conversationId)`，必要时消息发送者快照 |
| 发送文本/媒体/链接 | 乐观消息 + 成功替换 | `workbench.messages(conversationId)`、`workbench.workspace` |
| 重试投递/失败任务 | pending 态 | 对应管理列表与详情 Sheet query |
| 重置 token/删除应用 | 高危确认 | 应用列表、应用详情、相关绑定会话 |

禁止全局 invalidate；若一个 mutation 需要影响多个 query，必须在 feature `queries.ts` 集中列出，并用注释说明原因。

### 9.6 操作、键盘与命令面板

- 备注类单字段使用内联编辑（铅笔 → 输入框 + ✓/×，Enter 确认，Esc 取消）；多字段表单才用 Dialog。
- 危险操作分级：解绑用 AlertDialog；重置 token、删除应用需输入名称确认（type-to-confirm）。
- 表单 zod 错误首次 onBlur 展示，之后 onChange 即时更新；服务端字段错误映射回具体字段；Dialog 表单 dirty 时关闭需确认。
- 必须提供全局命令面板 `⌘K`：跳转会话、跳转页面、常用动作（同步通讯录、新建应用）。会话结果显示头像、名称、备注。
- 全站 `?` 呼出快捷键帮助；工作台 ↑/↓ 切会话；Esc 关闭浮层；可选 j/k 表格行导航。
- 所有可交互元素有可见 `focus-visible:ring`，Tab 顺序必须符合视觉顺序。
- 复制体系统一 `<CopyButton>`：图标变 ✓ 700ms；无图标反馈场景才额外 toast。

### 9.7 实时性与断线反馈

- 工作台 SSE 当前会话事件到达时直接追加消息流，不整页 invalidate；非当前会话只更新会话列表摘要与未读数。
- SSE 断开时顶栏显示黄色横幅“连接已断开，重连中…”，恢复后自动消失并静默补数据。
- 侧栏“运行观测”存在 failed 任务时显示红点，轮询 60s。

### 9.9 性能、可访问性与终验

- 路由级 code-splitting；聊天工作台作为首屏优先包。当前策略是工作台静态进入首屏包，管理型页面通过动态 `import()` 拆为独立 chunk。首屏 gzip 预算 < 300KB（不含字体），新增依赖前必须说明体积和必要性。
- 消息流 1000+ 条滚动 60fps；图片加载不造成相邻消息跳动；跳转指定消息可精确定位并 2s 高亮。
- 头像和媒体 `loading="lazy"`，失败降级占位；用户生成内容必须覆盖超长中文、超长英文无空格、emoji 名称回归测试。
- 使用 React Profiler 抽查：会话列表、消息气泡 memo 化；SSE 高频更新只影响相关 query/组件。
- Dialog/Sheet/Popover 焦点圈完整；图片有 alt；纯图标按钮有 aria-label；正文对比度 >= 4.5:1。
- 每个交付批次需要同数据前后截图；终验必须包含动效一致性、`⌘K`、消息分组与日期分隔、乐观发送与失败重试、断线横幅、1000 条滚动、超长文本、reduced-motion 降级。
